# 给 Claude Code 的说明 — 吠陀占星排盘 App

读完这份再开工。这个包里有三份文件，请按下面的角色使用它们，**有几处明确不能改**。

---

## 包内文件

1. **`vedic-mvp-spec.md`** — 产品技术规格。整体要做什么、技术栈、模块拆分、验收标准、精度坑都在这。**这是总纲，先读它。**

2. **`vedic_proto.py`** — 排盘计算引擎的**黄金参考实现**。
   - 已用 AstroSage 交叉验证过：九大行星星座、上升、宫位、Ketu、nakshatra、dasha 起算与当前大运，全部一致到角分级。
   - ⚠️ **计算逻辑不要改算法**。你的任务是把它包装进 Next.js（行星/上升计算放 Python 后端用 pyswisseph，dasha 逻辑可直接移植），不是重写它。
   - 三个致命设置必须保留，改了盘就错：① sidereal mode + Lahiri ayanamsa；② 整宫制(Whole sign)，不要 Placidus；③ Ketu = Rahu + 180。
   - 时区必须用 IANA 时区名(如 Asia/Shanghai)按出生日期解析历史夏令时，**不要硬编码 UTC 偏移**。部署时把 `tzdata` 写进依赖。

3. **`vedic_interpret.py`** — AI 解读模块，含**定稿的 system prompt**。
   - ⚠️ **system prompt 是产品的护城河，逐字保留，不要改写、不要精简、不要"优化"。** 它的术语尺度、文风规则、诚实择时原则都是反复打磨过的。
   - **已用真实 API 跑通验证**：解读准确(经本人核对)、文风正确(无破折号、宫位行话已转译、保留 Rahu/Ketu/dasha/落陷/nakshatra 并带人话解释)、十段结构完整。
   - **免费版输出 = 这套定稿的十段结构**(整体格局、性格、事业、工作环境、原生家庭、婚姻时机、过去择时、当前dasha、优势盲点清单、温暖总结)。
   - **付费深度版预留三类增量**(MVP 先不做，后续加)：① 每个维度的可执行建议；② 近一两年 antardasha 小运的细化窗口；③ 本命关键星宿的故事性解读。
   - **关键**：调用时必须把排盘引擎算出的「每颗行星的真实 nakshatra」和「完整 dasha 时间线」都传进 prompt（见 CHART 结构）。早期版本因为没传这两样，模型会自行编造星宿和历史大运，导致择时出错。这是必须遵守的。
   - 模型用 `claude-sonnet-4-6`。max_tokens 至少 4000（十段内容较长，低了会被截断）。
   - 调 Anthropic API 的方式见 `call_claude()`，后端代理，前端绝不暴露 key。
   - 调用时务必把"今天的日期"传进 `build_user_prompt(chart, today=...)`，模型靠它区分过去与未来。

## 实现顺序（务必按这个来）

1. 搭 Next.js + Tailwind 骨架（极简风：浅底、克制留白、单一强调色）。
2. 先把 Python 排盘后端跑通，用一组固定出生数据回归，对照 AstroSage 确认计算正确，**再往下做**。
3. 前端：出生信息输入表单（日期/时间/地点→地理编码出经纬度+时区名）。
4. 结果页：盘式图(SVG，North 或 South Indian 一种即可) + 关键信息表 + AI 解读。
5. 最后接 AI 解读，把排盘结果整理成 `vedic_interpret.py` 里 CHART 那样的结构喂给模型。

## MVP 不做（避免范围膨胀）

不做账号登录、不做 D9 分盘、不做当日 transit、不做合盘择日、不做支付、不做社区、不做小程序。先 Web 验证解读质量这一个核心假设。

## 验收

- 固定测试盘的上升/月亮星座/当前 dasha 与 AstroSage 一致。
- 全流程输入到出解读 < 10 秒。
- 手机浏览器正常显示（用户主要来自小红书，在手机上）。
- 解读输出符合 prompt 设定的文风：无破折号、宫位行话已翻译成生活语言、保留 Rahu/Ketu/dasha/落陷/nakshatra 等特色词并带人话解释。
